# Benchmark datasets

`smoke_testsrc` is generated by `scripts/prepare_assets.sh`. It is intended only
for installation checks and smoke runs.

Published benchmark runs use `mot17_uadetrac_public`. Download MOT17 and
UA-DETRAC from their official distribution points and place the extracted raw
sources under `data/videos/`:

- MOT17: <https://motchallenge.net/data/MOT17/>
- UA-DETRAC: <https://detrac-db.rit.albany.edu/>

The public benchmark manifest expects six normalized clips under
`data/benchmark/`. Prepare or verify them with:

```bash
.venv/bin/python scripts/prepare_benchmark_dataset.py --dataset mot17_uadetrac_public
.venv/bin/python scripts/check_dataset.py --dataset mot17_uadetrac_public --mode benchmark
```

The preparation script maps the manifest clips to these local raw sources:

- `MOT17/train/MOT17-02-FRCNN/img1` -> `data/benchmark/mot17_02.mp4`
- `MOT17/train/MOT17-04-FRCNN/img1` -> `data/benchmark/mot17_04.mp4`
- `MOT17/train/MOT17-09-FRCNN/img1` -> `data/benchmark/mot17_09.mp4`
- `DETRAC-Images/MVI_20011` -> `data/benchmark/uadetrac_mvi_20011.mp4`
- `DETRAC-Images/MVI_40152` -> `data/benchmark/uadetrac_mvi_40152.mp4`
- `DETRAC-Images/MVI_40714` -> `data/benchmark/uadetrac_mvi_40714.mp4`

Each generated clip is normalized to 1920x1080 at 30 FPS with H.264. Existing
clips whose SHA-256 matches `configs/datasets.yaml` are skipped. Missing or
mismatched clips are regenerated atomically and rejected if the resulting
checksum does not match the manifest. The script does not edit
`configs/datasets.yaml`; update the manifest intentionally if source material or
encoding settings change.

Use `--dry-run` to inspect regeneration work without writing files, and `--force`
to regenerate clips even when checksums currently match.

## Strict validation runbook

After telemetry adapter changes, run the full local strict matrix with:

```bash
.venv/bin/python scripts/validate_strict_telemetry_fix.py --clear-runtime-artifacts --policies all
```

This command prepares and validates `mot17_uadetrac_public`, builds local native
probe prerequisites, rebuilds derived DeepStream/Savant probe images, clears only
`.cache/savant` and `runs/strict_validation`, then runs `canonical_heterogeneous`
for all strict local adapters across all configured scheduler policies.

The runner rejects benchmark mode while a checksum is missing, uses a placeholder,
does not match the local file, or while native schema-v2 telemetry is incomplete
or malformed.
